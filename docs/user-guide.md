# PaperAgent 用户与操作手册

## 安装与启动

Windows 10/11 用户可直接双击便携包中的 `PaperAgent.exe`，也可运行 `install-user.ps1` 安装到当前用户并创建桌面/开始菜单快捷方式。源码模式执行 `.\scripts\start-local.ps1 Start`；脚本自动选择端口、健康检查和打开浏览器，停止与日志入口分别是 `Stop` 和 `Logs`。应用只监听 `127.0.0.1`，无需管理员权限。

首次运行页显示数据目录可写性、剩余磁盘、GPU、uv、Typst、Pandoc 和 TeX Live 状态。没有文本 Provider 时仍可导入、预览、管理资料与检查环境，但不能执行真实智能写作。可选依赖拒绝后不阻塞核心功能；点击安装时先显示来源、预计体积和目标路径，再等待明确确认。uv、Typst、Pandoc 通过 Windows Package Manager 当前用户范围安装；TeX Live 使用 TUG 官方网络安装器和 `scheme-full`，默认落在应用数据的 runtimes 目录，也可指定例如 `E:\App\PaperAgent-Runtimes\texlive`。安装在后台运行，状态与日志可追踪。

升级前，`install-user.ps1` 会安全停止旧实例，快照应用与用户数据，在暂存目录替换程序并对新版本执行 smoke；任何一步失败都会恢复旧程序和数据。`rollback-user.ps1` 可回到最近快照。卸载默认保留数据；`uninstall-user.ps1 -DataAction Export` 先导出，只有 `-DataAction Delete -ConfirmDataDeletion` 才删除用户数据。

## 基本工作流

1. 手动新建 Project，再在项目下新建一个或多个 Conversation；发送消息不会隐式创建项目。切换会话不打断其他后台 Run。
2. 输入自然语言需求或上传 PDF、DOCX、Markdown、代码与模板。
3. 在“Agent 计划”中并列查看原文、规范化需求、科学化候选与确认版 Requirement Spec。低置信度字段逐项接受、拒绝或补充。
4. 确认文体、大纲、字数预算、检索/实验/生图与输出格式。代码、安装和付费调用执行前会再次请求批准。
5. 在知识库查看来源、定位、可信等级、保密等级与引用资格。邮件、聊天、会议纪要默认只作过程资料。
6. 运行时消息卡实时显示公开阶段、尝试次数、事件数和耗时；活动中心可跨会话查看后台 Run。
7. 生成 Document IR，再输出 Markdown、DOCX、Typst/LaTeX 与 PDF。PDF 逐页 QA 后才成为已验证产物。
8. 在统一预览中批注或框选；系统把意见定位到 block/页码/区域，并生成局部 Repair Plan。

## 运行中追加要求（Steering）

- L0 独立问题走 sidecar，原 Run 继续。
- L1 状态询问直接读取公开时间线。
- L2 局部补充经确认后注入安全边界点。
- L3/L4 影响计划或既有结论时，从最早受影响 checkpoint 重规划/分支，保留稳定 artifact hash。
- L5 停止需确认；已产生内容保留并折叠，用户可展开查看。

应用重启时，无法证明安全重放的运行任务会进入 `paused/recovery_required`，不会把付费请求、文件覆盖或代码执行自动重发。

## 模型、隐私与翻译

设置页把“当前文本模型”“当前图片模型”“已保存配置”“新增配置”分开。每个新配置必须填写 URL、模型名和 API Key；保存后输入框恢复为空，模型切换只改变全局 binding，不复制 Key。支持 OpenAI、Claude、Gemini、DeepSeek、豆包、Xiaomi MiMo、Ollama 和自定义 OpenAI-compatible 服务。Key 由 Windows DPAPI 加密，不写普通数据库、`.env`、事件或日志。标准模式允许经批准外发；隐私控制模式先显示并脱敏外发内容；完全离线模式只阻止第三方 API，不影响本地导入、检索、预览和离线工具。

中英翻译在 Document IR 层工作，先保护公式、引用、代码和术语 token，再翻译正文并恢复 token。PDFMathTranslate-next 作为可选 AGPL 外部进程，不合并进核心发行包。

## 知识库、记忆与清理

项目库保存论文、技术文档、需求/设计、代码、会议、邮件、聊天、手册和 FAQ；全局库可保存个人长期复用资料。记忆采用可读 Markdown/YAML front matter、每日索引和 `MEMORY.md` manifest，支持查看、编辑、导出、禁用和清空；全局写入必须由用户明确触发，Key、凭据和原始敏感资料禁止进入记忆。会话短期记录由项目数据库与 append-only JSONL 保留。上下文按“安全规则→确认需求→任务状态→近期消息→相关记忆→证据→受保护摘要→工具状态”装配；压缩必须保留 Requirement 版本、locator、审批、Review Issue、工具调用配对和未知副作用。环境按依赖锁、Python、平台与硬件形成指纹复用，不按运行次数新建。

## 恢复与备份

恢复中心显示上次停止位置、已完成数、运行中/unknown 动作及下一步。付费 API 和代码执行启动后绝不自动恢复；用户查看 request id、可能费用和范围后选择继续或跳过。SQLite 使用 WAL 在线备份；迁移前备份、每日备份和最近 7 份滚动保留。恢复演练会校验 SHA-256 和 `PRAGMA quick_check`，项目可导出为 zip。

## 常见问题

- 没有 API Key：可完成导入、预览、知识管理、备份恢复和环境检测，真实智能写作暂不可用。
- 没有 Word/TeX Live：优先用 Typst 生成 PDF；DOCX 可直接生成，Word 转 PDF 路径不可用时会明确降级。
- GPU 跑不动仓库：能力分析器根据 README、锁文件、CUDA 要求和显存判断；不合理时阻断或建议 CPU/缩小规模，不盲目执行。
- 任务中断：重新打开恢复中心。`unknown` 不等于失败，尤其不能直接重发付费调用。
- 预览失败：仍显示元数据、扫描结果、失败原因和系统打开入口，不执行宏或嵌入脚本。
- 中文/空格路径：核心路径已纳入测试；若某个外部工具不支持，会在检测阶段报告而不是改写用户路径。

## 先预览、再导出和重新排版

首次要求报告但未指定格式时，聊天区先显示浏览器预览，同时后台保存 canonical revision。之后输入“将结果转换成 PDF”“再导出 Word 和 Markdown bundle”会复用该 revision 和已验证图片，不会重新撰写正文或重跑实验。成功文件出现在对应回复下方，可下载并加入右侧多标签预览；校验失败文件不会伪装成成功附件。

“字体改为宋体，正文小四”“把第一张图缩小”等请求会创建新 revision。全局字体和字号只改变样式 hash；定向图片调整使用稳定 block ID。若当前项目或会话存在多个可能目标，恢复中心会列出标题、revision 和来源，让用户选择后继续原任务，系统不会猜测。

只有明确输入“重新运行实验再导出”等肯定指令，并完成本地执行授权，系统才重跑必要实验节点；“不要重跑实验”“只重新排版”会作为硬约束。暂停、应用重启或安全故障后，从原任务的 checkpoint 恢复，先核对 execution/artifact ledger，已经成功的 Python、图片和交付文件不会重复生成。

## 封面个人信息与页眉页脚

首次可直接说：“封面标题为驻波实验报告，姓名为张三，学号为 2026001，学校为某大学，指导教师为李老师；封面不显示页眉页脚，正文页眉为大学物理实验报告，页脚为第 {page} 页 / 共 {pages} 页。”未提供的个人值不会被编造，系统会在计划确认前要求补充。

成文后可说：“基于刚才的报告，只修改文档呈现：学校改为某科技大学，删除班级，课程改为高级大学物理实验，页眉改为高级物理实验报告，导出 PDF 和 DOCX；不要重写正文，不要重跑实验。”该请求只创建 Presentation revision，不进入 Writer、Experiment 或 Image 节点。活动时间线只显示变更区域、字段数量和 hash，不显示姓名、学号等具体值。

DOCX 和 XeLaTeX PDF 原生支持封面、重复页眉页脚和动态页码。Markdown 可保存封面字段的可移植语义，但没有通用的物理页眉页脚；界面会显示能力降级说明，不会把浏览器预览效果冒充为 Markdown 文件能力。
